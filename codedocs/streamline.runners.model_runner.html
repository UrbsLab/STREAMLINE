Runner Class for running all the model jobs for cross-validation splits.

Parameters:

  • output_path ‚Äì path to output directory

  • experiment_name ‚Äì name of experiment (no spaces)

  • algorithms ‚Äì list of str of ML models to run

  • scoring_metric ‚Äì primary scikit-learn specified scoring metric used for hyperparameter optimization and permutation-based model feature importance evaluation, default=‚Äôbalanced_accuracy‚Äô

  • metric_direction ‚Äì direction to optimize the scoring metric in optuna, either ‚Äòmaximize‚Äô or ‚Äòminimize‚Äô, default=‚Äômaximize‚Äô

  • training_subsample ‚Äì for long running algos (XGB,SVM,ANN,KNN), option to subsample training set (0 for no subsample, default=0)

  • use_uniform_fi ‚Äì overrides use of any available feature importance estimate methods from models, instead using permutation_importance uniformly, default=True

  • n_trials ‚Äì number of bayesian hyperparameter optimization trials using optuna (specify an integer or None) default=200

  • timeout ‚Äì seconds until hyperparameter sweep stops running new trials (Note: it may run longer to finish last trial started) If set to None, STREAMLINE is completely replicable, but will take longer to run default=900 i.e. 900 sec = 15 minutes default save_plots: export optuna-generated hyperparameter sweep plots, default False

  • do_lcs_sweep ‚Äì do LCS hyper-param tuning or use below params, default=False

  • lcs_nu ‚Äì fixed LCS nu param (recommended range 1-10), set to larger value for data with less or no noise, default=1

  • lcs_iterations ‚Äì fixed LCS number of learning iterations param, default=200000

  • lcs_n ‚Äì fixed LCS rule population maximum size param, default=2000

  • lcs_timeout ‚Äì seconds until hyperparameter sweep stops for LCS algorithms, default=1200
